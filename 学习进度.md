## 📝 学习心得

### Day 01 心得
今天学习了Python基础语法，重点掌握了：
1. 列表和字典是AI开发中最常用的数据结构
2. JSON处理是API数据交互的核心技能
3. 函数的参数类型（默认参数、*args、**kwargs）在配置管理中很实用

### Day 02 心得
今天学习了面向对象编程，重点掌握了：
1. 类与对象是Python进阶的核心，理解`__init__`、`__str__`等特殊方法
2. 继承与多态让代码更具扩展性，Mixin模式灵活组合功能
3. 装饰器在框架开发中非常常用，`@property`、`@staticmethod`、`@classmethod`各有用途
4. 自定义异常让错误处理更清晰，上下文管理器简化资源管理

### Day 03 心得
今天学习了文件操作与正则表达式，重点掌握了：
1. `with`语句确保文件正确关闭，是文件操作的最佳实践
2. `pathlib.Path`比传统`os.path`更直观，推荐使用`/`操作符拼接路径
3. 正则表达式是文本处理的利器，分组捕获可以提取结构化数据
4. 数据验证（邮箱、手机号）是AI开发中常见的需求

### Day 04 心得
今天学习了模块、虚拟环境与单元测试，重点掌握了：
1. `__name__ == "__main__"` 是Python模块的入口判断机制，被导入时不会执行主逻辑
2. 虚拟环境是AI项目的标配，每个项目独立隔离依赖，避免版本冲突
3. `requirements.txt` 是团队协作和项目部署的基础，养成 `pip freeze` 的习惯
4. 单元测试让代码更可靠，`setUp/tearDown` 保证每个测试用例的独立性

### Day 05 心得
今天学习了HTTP请求与数据处理，重点掌握了：
1. requests 库是调用 AI API 的核心工具，Session 复用可以统一管理 Headers 和连接
2. 状态码处理是生产环境必备，429限流和500错误都需要重试机制
3. 列表推导式是数据清洗的利器，一行代码完成过滤+转换
4. 模拟 OpenAI API 的请求/响应结构，为后续真实调用打好基础

### Day 06 心得
今天学习了异步编程，重点掌握了：
1. 同步 vs 异步的本质区别：异步不是多线程，而是在等待IO时切换执行其他任务
2. `async/await` 语法让异步代码像同步一样易读
3. `asyncio.gather` 并发调用5个请求，耗时从0.5s降到0.1s，效果显著
4. 批量调用 LLM API 时异步是必选项，可以大幅提升吞吐量

### Day 07 心得
今天学习了Git基础并完成了Week01复盘，重点掌握了：
1. Git三区模型（工作区→暂存区→仓库）是理解所有Git操作的基础
2. `.gitignore` 中 `.env` 和 `venv/` 是AI项目最重要的两条规则，API Key绝不能提交
3. 标准的AI项目目录结构让代码更易维护，`src/`、`tests/`、`data/` 各司其职
4. Week01 综合复盘将7天知识串联，从数据结构到异步调用，构成了完整的AI开发基础

### Day 08 心得
今天学习了LangChain架构与LLM基础，重点掌握了：
1. LangChain四大核心组件（LLM、Chain、Agent、Memory）各自的职责与协作关系
2. ChatOpenAI是LangChain中调用OpenAI的主要方式，invoke()方法支持字符串和消息列表两种输入
3. temperature参数控制输出创造性，0.7是平衡点，0.0最确定性，2.0最随机
