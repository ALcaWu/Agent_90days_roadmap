## 一、模块与包管理

### 1.1 模块的概念

**模块 = 一个.py文件**，包含可重用的代码（函数、类、变量）。

### 1.2 import的四种方式

```python
# 方式1：导入整个模块
import math
math.sqrt(16)

# 方式2：导入特定对象
from math import sqrt, pi

# 方式3：起别名（AI开发常用）
import numpy as np
import pandas as pd

# 方式4：导入所有（不推荐）
from math import *  # 污染命名空间
```

### 1.3 `__name__ == "__main__"` 机制

```python
def main():
    print("主程序逻辑")

if __name__ == "__main__":
    main()  # 只有直接运行时执行
```

| 场景 | `__name__`值 |
|------|-------------|
| 直接运行文件 | `"__main__"` |
| 被import导入 | 模块名（文件名） |

### 1.4 包的结构

```
mypackage/
├── __init__.py      # 包标识文件
├── module_a.py
└── subpackage/
    ├── __init__.py
    └── module_b.py
```

### 1.5 AI开发常用标准库

| 模块 | 用途 |
|------|------|
| `os` | 文件路径、环境变量 |
| `sys` | Python解释器信息 |
| `json` | API数据交互 |
| `pathlib` | 现代路径操作 |
| `typing` | 类型提示（AI开发必备） |
| `logging` | 日志记录 |
| `re` | 正则表达式 |

## 二、虚拟环境与pip

### 2.1 为什么需要虚拟环境？

```
问题：项目A需要langchain 0.1.0，项目B需要langchain 0.2.0
解决：每个项目独立的虚拟环境，隔离依赖
```

### 2.2 venv常用命令

```bash
# 创建虚拟环境
python -m venv venv

# 激活（Windows）
venv\Scripts\activate

# 激活（macOS/Linux）
source venv/bin/activate

# 退出
deactivate
```

### 2.3 pip常用命令

```bash
pip install langchain          # 安装
pip install langchain==0.1.0   # 指定版本
pip list                       # 查看已安装
pip freeze > requirements.txt  # 导出依赖
pip install -r requirements.txt  # 从文件安装
pip uninstall langchain        # 卸载
```

### 2.4 requirements.txt格式

```txt
# AI开发常用依赖
langchain>=0.1.0
langchain-openai>=0.0.5
chromadb>=0.4.0
python-dotenv>=1.0.0
pydantic>=2.0.0
```

### 2.5 最佳实践

1. 每个项目一个虚拟环境
2. 将 `venv/` 加入 `.gitignore`
3. 使用 `requirements.txt` 管理依赖
4. 区分开发依赖和生产依赖

## 三、单元测试基础

### 3.1 unittest框架结构

```python
import unittest

class TestMyFunction(unittest.TestCase):
    
    def setUp(self):
        """每个测试方法前执行"""
        pass
    
    def test_something(self):
        """测试方法（必须以test_开头）"""
        self.assertEqual(1 + 1, 2)
    
    def tearDown(self):
        """每个测试方法后执行"""
        pass
```

### 3.2 常用断言方法

| 方法 | 说明 |
|------|------|
| `assertEqual(a, b)` | a == b |
| `assertNotEqual(a, b)` | a != b |
| `assertTrue(x)` | x is True |
| `assertFalse(x)` | x is False |
| `assertIn(a, b)` | a in b |
| `assertIsNone(x)` | x is None |
| `assertRaises(Error)` | 期望抛出异常 |
| `assertAlmostEqual(a, b, places=4)` | 浮点数近似相等 |

### 3.3 运行测试

```bash
# 运行单个文件
python test_file.py

# 详细模式
python -m unittest test_file.py -v

# 运行所有测试
python -m unittest discover
```

### 3.4 AI开发中的测试场景

- 测试Agent输出是否符合预期格式
- 测试Chain的输入输出
- 测试工具函数正确性
- 重构代码时确保功能不退化
